# Project Documentation

This folder contains all project documentation. Use the numbered sections for consistent ordering.

## Structure
- 00-overview: Vision, scope, glossary, stakeholders, and domain context.
- 01-architecture: System design, Clean Architecture rules, service boundaries.
- 02-services: Per-service documentation and ownership.
- 03-api: API specs, contracts, and error models.
- 04-data: Data models, schemas, and migrations.
- 05-infra: Environments, deployment, CI/CD, and observability.
- 06-security: Auth, RBAC, privacy, and audit trails.
- 07-ops: Runbooks, SLOs, incident response.
- 08-frontend: Web app architecture and UI conventions.
- 09-mobile: Flutter architecture and offline sync design.
- 10-testing: Test strategy and quality gates.
- 11-compliance: Compliance requirements and checklists.
- 12-decisions: ADRs (architecture decision records).
- 13-roadmap: Milestones and phase plans.
